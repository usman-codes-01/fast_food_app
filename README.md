# Campus Bites

A full-stack campus canteen ordering app built with **Flutter**, **GetX**, and **Firebase**. Students browse the menu, place orders, and track them live; kitchen staff manage incoming orders and the menu from a dedicated admin dashboard.

> Built as an end-to-end product: real authentication, real-time database, role-based UI, dark mode, and an OTP-based handoff flow.

---

## Tech Stack

| Layer | Technology |
| --- | --- |
| Framework | Flutter (Dart 3.9+) |
| State Management | GetX (reactive controllers, routing, dependency injection) |
| Authentication | Firebase Authentication (Email / Password) |
| Database | Cloud Firestore (real-time streams) |
| Architecture | MVC-style separation — `models`, `controllers`, `screens`, `widgets` |
| Theming | Material 3 with custom light / dark themes |

---

## Features

### Student
- Splash → Onboarding → Login / Register flow
- Browse menu by **category** with **live search**
- Cart with quantity controls and running total
- Place order and receive a unique **4-digit pickup OTP**
- **Live order status** tracking (Pending → Cooking → Ready)
- Profile screen with **dark mode** toggle and logout

### Admin / Kitchen
- Live kitchen display board with three lanes: **Pending**, **Cooking**, **Ready**
- Multi-item order cards with customer details
- **OTP verification** at handoff to prevent wrong pickups
- Full **menu CRUD** — add, edit, delete items, toggle in-stock status
- Real-time updates across all connected devices via Firestore streams

## Getting Started

### Prerequisites
- Flutter SDK 3.9+
- A Firebase project (free Spark plan is enough)
- Node.js (for the Firebase CLI)

### 1. Clone & install dependencies
```bash
git clone <your-fork-url>
cd fast_food_app
flutter pub get
```

### 2. Configure Firebase
The repo intentionally ships **without** Firebase keys. On first launch the app will show a *"Firebase not configured"* screen. To wire it up:

```bash
# One-time CLI install
dart pub global activate flutterfire_cli
npm install -g firebase-tools

# Log in and configure
firebase login
flutterfire configure
```

`flutterfire configure` will:
- Generate the real `lib/firebase_options.dart` (overwriting the stub)
- Place `android/app/google-services.json`
- Place `ios/Runner/GoogleService-Info.plist` (if iOS is selected)

### 3. Enable Firebase services
In the Firebase Console for your project, enable:
- **Authentication → Sign-in method → Email/Password**
- **Cloud Firestore** (start in test mode for development)

### 4. Run the app
```bash
flutter run
```
